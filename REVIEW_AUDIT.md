# REVIEW AUDIT

> 审计时间：2026-04-16  
> 范围：根目录 Markdown 文档与当前代码实现的一致性

## 本轮审计结论

本轮已将根目录文档统一收敛为“一个入口文档 + 多个单一职责专题文档”的结构，主要解决了 3 个问题：

- 根目录文档之间职责重复
- README、规划、联调文档存在信息分散
- 部分根目录文档缺少与当前实现的交叉引用

当前根目录文档的角色已经明确：

- `README.md`：入口文档，负责项目总览、启动、入口、索引
- `Flash-Sale-System架构与接口规范.md`：负责模块边界、链路、接口分层
- `前后端交互规范.md`：负责页面访问规则、前端 API 口径、联调清单
- `已实现技术栈解析与使用说明.md`：负责技术栈与项目内落地方式
- `前端完整化待补充后端接口清单.md`：负责仍未完成但值得继续补的后端能力
- `ai-service-优化方向.md`：负责 AI 模块优化项与优先级
- `planning.md`：负责阶段规划和推进顺序
- `REVIEW_AUDIT.md`：负责记录审计结论与维护规则

## 本轮已同步的关键信息

- 前端当前实际路由包含 `/app/assistant`
- 后端服务默认端口和模块边界已同步到根目录文档
- 前端 API 调用口径已按当前 `flash-sale-ui/src/api` 收敛
- 秒杀链路中 Redis、Lua、RabbitMQ 的职责边界已重新明确
- AI 模块最近修复的 Redis 正确性问题已纳入文档现状

## 当前残留风险

### 1. Maven Wrapper 仍缺失

仓库仍没有 `mvnw`。不同机器上的编译和启动体验，仍依赖本机 Maven 或 IDE。

### 2. `application-local.yml` 仍可能覆盖 Nacos

配置优先级仍需谨慎，联调时要明确最终生效值来自哪里。

### 3. 编译验证受环境限制

在当前沙箱环境下，Maven 依赖无法从外网下载，因此不能用完整编译来给文档改动做全量验证。

## 文档维护规则

- 项目入口、启动方式、端口变化时，更新 `README.md`
- 模块边界、链路、接口层次变化时，更新 `Flash-Sale-System架构与接口规范.md`
- 前端路由、API 调用方式、联调口径变化时，更新 `前后端交互规范.md`
- 已完成技术方案变化时，更新 `已实现技术栈解析与使用说明.md`
- 待补后端能力变化时，更新 `前端完整化待补充后端接口清单.md`
- AI 规划变化时，更新 `ai-service-优化方向.md`
- 研发优先级和阶段路线变化时，更新 `planning.md`

## 审计结论

根目录文档当前已经可以作为一套统一入口使用，后续只要按上面的职责分工维护，就不容易再次出现“多个文档同时讲一件事、但口径不一致”的问题。
