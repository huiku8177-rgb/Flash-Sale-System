# 已实现技术栈解析与使用说明

## 1. 后端基础框架

### Java 17

用途：

- 作为后端统一运行时版本
- 支撑 Spring Boot 3.x 与 Spring Cloud 2023

当前使用方式：

- `flash-sale-serve` 所有服务统一使用 Java 17

### Spring Boot 3.2.5

用途：

- 提供 Web、配置、依赖注入与自动装配能力

当前使用方式：

- `auth-service` 负责认证与地址管理
- `product-service` 负责普通商品、购物车与普通下单入口
- `order-service` 负责订单查询、支付、取消与内部建单
- `seckill-service` 负责秒杀链路
- `gateway` 负责统一入口

### Spring Cloud 2023.0.1

用途：

- 微服务治理与服务间通信基础

当前使用方式：

- 通过 Gateway 实现路由
- 通过 OpenFeign 实现服务间同步调用

### Spring Cloud Alibaba 2023.0.1.0

用途：

- 提供 Nacos 注册中心与配置中心支持

当前使用方式：

- 微服务注册到 Nacos
- `application.yml` 通过 `spring.config.import` 从 Nacos 拉取共享配置和服务配置

## 2. 网关与服务治理

### Spring Cloud Gateway

用途：

- 统一网关入口
- 鉴权
- 路由转发
- 限流
- Swagger 聚合

当前使用方式：

- 对外统一暴露 `http://localhost:8080`
- 支持白名单路径放行
- 自动移除外部伪造的 `X-User-Id`
- 鉴权成功后向下游写入真实用户身份
- 聚合各服务 OpenAPI 文档

### OpenFeign

用途：

- 微服务之间同步调用

当前使用方式：

- `product-service` 调用 `order-service` 创建普通订单
- 服务间透传部分请求头，降低上下文丢失

## 3. 认证与安全

### JWT

用途：

- 无状态登录认证

当前使用方式：

- 登录成功后由认证服务签发 Token
- Gateway 统一校验 Token
- 下游业务服务不重复解析 JWT

### BCrypt

用途：

- 密码加密存储与比对

当前使用方式：

- 注册时加密密码
- 登录时校验密码
- 修改密码时校验旧密码并重新编码

## 4. 数据访问与基础设施

### MyBatis

用途：

- 实现持久层访问

当前使用方式：

- `product-service`、`order-service`、`seckill-service` 以 Mapper + XML 为主
- `auth-service` 也基于 MyBatis 进行用户与地址管理

### MySQL

用途：

- 持久化业务数据

当前核心表：

- `user`
- `user_address`
- `product`
- `cart_item`
- `seckill_product`
- `normal_order`
- `normal_order_item`
- `seckill_order`

### Redis

用途：

- 秒杀库存控制
- 秒杀结果缓存
- 部分限流与高并发场景支撑

当前使用方式：

- 秒杀库存预热
- Lua 原子扣减与防重
- 秒杀结果缓存与回查

### RabbitMQ

用途：

- 秒杀异步建单

当前使用方式：

- `seckill-service` 投递秒杀建单消息
- `order-service` 消费消息并落库

### Nacos

用途：

- 服务注册发现
- 配置中心

当前使用方式：

- 服务注册与发现
- Nacos 配置中心维护共享配置和服务私有配置
- 本地仍保留 `application-local.yml` 作为兜底

## 5. 接口文档与调试

### SpringDoc OpenAPI

用途：

- 生成 OpenAPI 文档

当前使用方式：

- 各服务输出自己的 `/v3/api-docs`
- Gateway 聚合成统一文档入口

### Swagger UI

用途：

- 在线调试接口

当前入口：

- `http://localhost:8080/swagger-ui.html`

## 6. 前端技术栈

### Vue 3

用途：

- 构建商城前端页面

当前使用方式：

- 登录 / 注册页
- 商城首页
- 秒杀页
- 购物车页
- 结算页
- 订单中心
- 账户资料与密码安全页

### Vue Router 4

用途：

- 前端路由管理

当前使用方式：

- 采用 `createWebHashHistory()`
- 支持登录守卫与访客页守卫
- 登录页与注册页分路由

### Element Plus

用途：

- 表单、弹窗、抽屉、消息提示等通用组件

当前使用方式：

- 登录注册表单
- 商品详情抽屉
- 订单详情弹窗
- 提示与交互反馈

### Axios

用途：

- 前后端 HTTP 通信

当前使用方式：

- 按业务域拆分 API 文件
- 请求拦截器自动注入 Bearer Token
- 响应拦截器统一处理 `401`

### Vite 6

用途：

- 前端开发与构建

当前使用方式：

- `npm run dev`
- `npm run build`

## 7. 当前工程化现状

### 配置管理

当前采用：

- `application.yml`：服务名、Nacos 地址与配置导入
- `application-local.yml`：本地兜底
- Nacos：共享配置与服务私有配置

### 日志

当前已经在多个核心控制器、网关过滤器和异步链路上补充了 `Slf4j` 日志。

### 观测与调试

当前已经具备：

- Swagger 聚合
- 请求链路基础日志
- 鉴权与限流基础能力

## 8. 适合继续补强的工程项

当前最值得继续做的工程化工作：

- 增加更完整的单元测试和集成测试
- 统一日志格式并显式输出 `requestId`
- 补充关键业务指标监控
- 补充后台管理与运营配置能力

## 9. 技术栈结论

这套项目目前已经不是单一的秒杀接口原型，而是一套具备以下特点的轻量商城系统：

- 普通商品链路与秒杀链路并存
- 微服务边界明确
- 前后端可以联调演示
- Nacos 已接入注册中心与配置中心
- 具备较好的继续扩展空间
