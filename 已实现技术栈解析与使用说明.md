# 已实现技术栈解析与使用说明

## 1. Java 17

用途：
- 作为后端统一语言版本与运行时
- 支撑 Spring Boot 3.x 与 Spring Cloud 2023 生态

当前使用方式：
- 后端多模块统一使用 Java 17
- `common`、`gateway`、`auth-service`、`product-service`、`seckill-service`、`order-service` 保持一致版本

## 2. Spring Boot

用途：
- 构建独立业务服务
- 提供 Web、配置、依赖注入与自动装配能力

当前使用方式：
- `auth-service` 负责认证链路
- `product-service` 负责普通商品、购物车、普通下单入口
- `seckill-service` 负责秒杀商品与秒杀流程
- `order-service` 负责订单查询、模拟支付与内部建单
- `gateway` 负责统一入口

## 3. Spring Cloud Gateway

用途：
- 统一入口
- 路由转发
- 网关鉴权
- Swagger 聚合

当前使用方式：
- 统一拦截 `Authorization: Bearer <token>`
- 支持“方法 + 路径”白名单规则
- 对外只暴露网关入口，不让前端直连微服务
- 清洗客户端伪造的 `X-User-Id`
- 统一向下游补充真实 `X-User-Id`
- 聚合 `auth-service`、`product-service`、`seckill-service`、`order-service` 文档

## 4. OpenFeign

用途：
- 微服务之间的同步调用

当前使用方式：
- `product-service` 通过 Feign 调用 `order-service`
- 当前主要用于普通订单内部建单与按订单号补查
- 内部接口不对前端开放

## 5. JWT + RSA

用途：
- 无状态认证
- 网关统一验签

当前使用方式：
- 登录成功后生成 JWT
- 网关统一解析与校验 JWT
- JWT 工具和密钥配置沉淀在公共模块与网关配置中

## 6. Spring Security Crypto / BCrypt

用途：
- 密码加密存储
- 登录密码匹配

当前使用方式：
- 注册时加密密码
- 登录时比对密码
- 修改密码时校验旧密码并重新编码新密码

## 7. MyBatis

用途：
- 数据访问层实现

当前使用方式：
- `auth-service` 主要使用注解 Mapper
- `product-service`、`seckill-service`、`order-service` 主要使用接口 + XML
- 购物车、普通订单、秒杀订单均通过 MyBatis 实现查询与写入

## 8. MySQL

用途：
- 存储用户、商品、购物车与订单数据

当前核心数据表：
- `user`
- `product`
- `cart_item`
- `seckill_product`
- `normal_order`
- `normal_order_item`
- `seckill_order`

## 9. Redis

用途：
- 秒杀库存控制
- 秒杀结果缓存
- 高并发场景削峰

当前使用方式：
- 秒杀商品库存预热
- Lua 原子扣减与防重控制
- 秒杀结果优先从 Redis 查询
- Redis 结果过期后再回查数据库兜底

## 10. RabbitMQ

用途：
- 秒杀成功后的异步建单

当前使用方式：
- `seckill-service` 负责发送秒杀建单消息
- `order-service` 负责消费消息并创建秒杀订单
- 用消息队列缓冲秒杀高峰请求

## 11. Nacos

用途：
- 服务注册与发现

当前使用方式：
- 网关通过 `lb://service-name` 转发请求
- OpenFeign 通过服务名发现 `order-service`
- 所有服务注册在同一 Nacos 命名空间下

## 12. SpringDoc OpenAPI / Swagger UI

用途：
- 生成接口文档
- 提供在线调试入口

当前使用方式：
- 各业务服务生成自己的 OpenAPI 文档
- 网关统一聚合到 `http://localhost:8080/swagger-ui.html`
- 接口文案统一改为简体中文
- 对外统一展示 `Authorization: Bearer <token>`
- 通过 `@Server(url = "/")` 配合网关入口进行 Try it out

## 13. Vue 3

用途：
- 构建前端页面与状态交互

当前使用方式：
- 登录页
- 商城首页
- 秒杀会场页
- 购物车页
- 独立结算页
- 个人中心页

## 14. Vue Router

用途：
- 前端页面路由管理

当前使用方式：
- 使用 `createWebHashHistory()`
- 当前主要路由：
  - `/login`
  - `/app/home`
  - `/app/flash`
  - `/app/cart`
  - `/app/profile`
  - `/checkout`
- 匿名可访问首页与秒杀页
- 购物车、个人中心、结算页要求登录

## 15. Element Plus

用途：
- 表单、按钮、消息提示、抽屉与弹窗等基础组件

当前使用方式：
- 登录注册表单
- 商品详情抽屉
- 订单详情弹窗
- 各类消息提示与交互反馈

## 16. Axios

用途：
- 前后端 HTTP 通信

当前使用方式：
- 前端统一通过 Axios 请求网关
- 请求拦截器自动注入 `Bearer Token`
- 响应拦截器统一处理 `401`
- 普通商品、购物车、下单、秒杀、订单查询均使用统一请求封装

## 17. Vite

用途：
- 前端开发与构建工具

当前使用方式：
- `npm run dev`
- `npm run build`

说明：
- 当前前端构建可通过
- 仍存在默认的包体积告警，但不影响功能演示

## 18. 当前技术栈结论

这套项目现在已经不是早期的“秒杀后端原型”，而是具备以下闭环能力的轻量商城系统：

- 登录与注册
- 普通商品浏览
- 后端购物车
- 立即购买与独立结算页
- 普通订单创建、查询与模拟支付
- 秒杀商品浏览
- 秒杀请求与轮询结果
- 秒杀异步建单
- 秒杀订单查询与模拟支付
- Swagger 网关聚合联调

如果继续完善，最值得优先补的仍然是：
- 地址管理
- 购物车结算预览
- 订单取消 / 超时关闭
- 统一订单聚合能力
