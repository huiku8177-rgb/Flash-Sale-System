# 已实现技术栈解析与使用说明

## 1. Java 17

用途：

- 作为后端主语言与运行时
- 支撑 Spring Boot 3.x 生态

当前使用方式：

- 多模块后端统一以 Java 17 为目标版本
- `common`、`gateway`、`auth-service`、`product-service`、`seckill-service`、`order-service` 共享同一套基础语言版本

## 2. Spring Boot

用途：

- 构建独立业务服务
- 提供 Web、配置、依赖注入与启动能力

当前使用方式：

- `auth-service` 负责认证
- `product-service` 负责普通商品与普通下单入口
- `seckill-service` 负责秒杀商品与秒杀流程
- `order-service` 负责订单查询、支付与内部建单
- `gateway` 负责统一入口

## 3. Spring Cloud Gateway

用途：

- 统一路由
- 鉴权前置
- 聚合文档入口

当前使用方式：

- 统一拦截 Bearer Token
- 白名单请求直接放行
- 对非白名单请求解析 JWT
- 透传 `X-User-Id`
- 聚合 Swagger UI
- 路由到 `/auth/**`、`/product/**`、`/seckill/**`、`/seckill-product/**`、`/order/**`

## 4. OpenFeign

用途：

- 普通商品服务到订单服务的同步调用

当前使用方式：

- `product-service` 通过 Feign 调用 `order-service`
- 用于普通商品建单后的订单落库
- 内部调用接口不对前端开放

## 5. JWT + RSA

用途：

- 无状态认证
- 便于网关统一验签

当前使用方式：

- 登录成功后签发 JWT
- 网关统一解析 JWT
- JWT 工具沉淀在 `common` 模块

## 6. Spring Security Crypto / BCrypt

用途：

- 密码加密存储
- 登录密码比对

当前使用方式：

- 注册时 `encode`
- 登录时 `matches`
- 修改密码时校验旧密码

## 7. MyBatis

用途：

- 数据访问层

当前使用方式：

- `auth-service` 主要使用注解方式 Mapper
- `product-service`、`seckill-service`、`order-service` 主要使用接口 + XML
- 普通订单与秒杀订单都通过 MyBatis 完成查询和写入

## 8. MySQL

用途：

- 存储用户、普通商品、秒杀商品、普通订单、秒杀订单

当前使用表：

- `user`
- `product`
- `seckill_product`
- `normal_order`
- `normal_order_item`
- `seckill_order`

## 9. Redis

用途：

- 秒杀库存
- 秒杀用户资格
- 秒杀结果缓存

当前使用方式：

- 秒杀库存预热
- Lua 原子扣减与防重
- 秒杀排队结果缓存
- 秒杀结果查询优先查 Redis，过期后回查数据库

## 10. RabbitMQ

用途：

- 秒杀成功后的异步建单

当前使用方式：

- 秒杀请求通过 MQ 解耦下单
- `order-service` 负责消费消息创建秒杀订单
- 支持失败补偿与死信兜底

## 11. Nacos

用途：

- 服务注册与发现

当前使用方式：

- Gateway 通过 `lb://service-name` 发现下游服务
- OpenFeign 通过服务名查找 `order-service`
- 所有服务注册到同一个 Nacos namespace

## 12. SpringDoc OpenAPI / Swagger UI

用途：

- 生成 API 文档
- 网关统一聚合调试

当前使用方式：

- 每个业务服务生成自己的 OpenAPI 文档
- 网关统一聚合到 `http://localhost:8080/swagger-ui.html`
- 文档口径统一为 `Authorization: Bearer <token>`

## 13. Vue 3

用途：

- 构建前端页面与状态交互

当前使用方式：

- 登录页
- 首页
- 秒杀页
- 购物车页
- 个人中心页

## 14. Vue Router

用途：

- 页面路由管理

当前使用方式：

- `/login`
- `/app/home`
- `/app/flash`
- `/app/cart`
- `/app/profile`

## 15. Element Plus

用途：

- 表单、按钮、弹窗、标签等组件

当前使用方式：

- 登录注册表单
- 商品详情抽屉
- 订单详情弹窗
- 首页与购物车交互组件

## 16. Axios

用途：

- 前后端 HTTP 通信

当前使用方式：

- 统一网关客户端
- 请求拦截器自动注入 Bearer Token
- 响应拦截器统一处理 `Result<T>` 与 `401`
- 普通商品下单当前也默认走网关地址

## 17. Vite

用途：

- 前端开发与构建工具

当前使用方式：

- `npm run dev`
- `npm run build`

## 18. 当前技术栈结论

这套项目现在已经不是早期的“秒杀原型”，而是具备了以下真实闭环：

- 登录注册
- 普通商品查询
- 普通商品下单
- 普通订单查询与支付
- 秒杀商品查询
- 秒杀请求与轮询结果
- 秒杀异步建单
- 秒杀订单查询与支付
- Swagger 聚合联调
- 前端完整页面联调

后续如果继续推进，最值得继续补的是：

- 后端购物车服务
- 地址管理
- 统一订单聚合接口
- 秒杀场次与活动运营化
