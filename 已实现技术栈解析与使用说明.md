# 已实现技术栈解析与使用说明

## 1. Java 17

用途：

- 作为后端主语言与运行时
- 支撑 Spring Boot 3.x 生态

当前使用方式：

- 多模块后端统一以 Java 17 为目标版本
- `common`、`gateway`、`auth-service`、`seckill-service`、`order-service` 共享同一套基础语言版本

## 2. Spring Boot

用途：

- 构建各个独立业务服务
- 提供 Web、配置、依赖注入与启动能力

当前使用方式：

- `auth-service` 负责认证
- `seckill-service` 负责商品与秒杀
- `order-service` 负责订单
- `gateway` 负责统一入口

## 3. Spring Cloud Gateway

用途：

- 统一路由
- 鉴权前置
- 透传用户身份

当前使用方式：

- 统一拦截 Bearer Token
- 对白名单请求直接放行
- 对非白名单请求解析 JWT
- 透传 `X-User-Id`
- 路由到 `/auth/**`、`/order/**`、`/seckill/**`、`/seckill-product/**`、`/product/**`

## 4. JWT + RSA

用途：

- 无状态认证
- 便于网关统一验签

当前使用方式：

- 登录成功后签发 JWT
- 网关统一解析 JWT
- JWT 工具沉淀在 `common` 模块

## 5. Spring Security Crypto / BCrypt

用途：

- 密码加密存储
- 登录密码比对

当前使用方式：

- 注册时 `encode`
- 登录时 `matches`
- 修改密码时也会使用旧密码校验

## 6. MyBatis

用途：

- 数据访问层

当前使用方式：

- `auth-service` 主要使用注解方式 Mapper
- `seckill-service` 与 `order-service` 主要使用接口 + XML
- 普通订单与秒杀订单都通过 MyBatis 完成查询和写入

## 7. MySQL

用途：

- 存储用户、商品、秒杀订单、普通订单

当前使用表：

- `user`
- `product`
- `seckill_product`
- `seckill_order`
- `normal_order`
- `normal_order_item`

## 8. Redis

用途：

- 秒杀库存
- 用户秒杀状态
- 秒杀结果缓存

当前使用方式：

- 秒杀库存预热
- Lua 原子扣减
- 秒杀排队结果缓存
- 秒杀结果查询优先查 Redis，过期后回查数据库

## 9. RabbitMQ

用途：

- 秒杀成功后的异步建单

当前使用方式：

- 秒杀请求通过 MQ 解耦下单
- `order-service` 负责消费消息创建秒杀订单

## 10. Nacos

用途：

- 服务注册与发现

当前使用方式：

- Gateway 通过 `lb://service-name` 发现下游服务
- 多个服务都注册到同一个 Nacos namespace

## 11. Vue 3

用途：

- 构建前端页面与状态交互

当前使用方式：

- 登录页
- 首页
- 秒杀页
- 购物车页
- 个人中心页

## 12. Vue Router

用途：

- 页面路由管理

当前使用方式：

- `/login`
- `/app/home`
- `/app/flash`
- `/app/cart`
- `/app/profile`

## 13. Element Plus

用途：

- 表单、按钮、弹窗、抽屉、标签等组件

当前使用方式：

- 登录注册表单
- 商品详情抽屉
- 订单详情弹窗
- 首页与购物车交互组件

## 14. Axios

用途：

- 前后端 HTTP 通信

当前使用方式：

- 统一网关客户端
- 请求拦截器自动注入 Bearer Token
- 响应拦截器统一处理 `Result<T>` 与 `401`

## 15. Vite

用途：

- 前端开发与构建工具

当前使用方式：

- `npm run dev`
- `npm run build`

## 16. 当前技术栈结论

这套项目现在不是“只有认证和接口骨架”，而是已经具备了以下可运行链路：

- 登录注册
- 普通商品查询
- 秒杀商品查询
- 秒杀请求与轮询结果
- 秒杀异步建单
- 普通订单创建与支付
- 秒杀订单支付
- 前端完整联调页面

后续如果继续推进，最值得继续补的是：

- 真实购物车后端
- 地址管理
- 统一订单聚合接口
- 首页装修与搜索推荐
