# 已实现技术栈解析与使用说明

## 文档定位

本文档只说明两件事：

1. 这个项目用了什么技术
2. 这些技术在当前代码里是怎么落地的

## 后端基础框架

| 技术 | 当前版本 | 项目内用途 |
| --- | --- | --- |
| Java | 17 | 所有后端服务统一运行时 |
| Spring Boot | 3.2.5 | Web、配置、自动装配、依赖注入 |
| Spring Cloud | 2023.0.1 | 网关、服务治理、OpenFeign 体系 |
| Spring Cloud Alibaba | 2023.0.1.0 | Nacos 注册中心与配置中心 |

落地情况：

- `flash-sale-serve` 是 Maven 多模块工程
- 各业务服务都基于 Spring Boot 独立启动
- `gateway`、`auth-service`、`product-service`、`order-service`、`seckill-service`、`ai-service` 已拆分完成

## 服务治理与网关

### Spring Cloud Gateway

当前用于：

- 统一入口
- JWT 鉴权
- 白名单放行
- 用户身份透传
- 路由分发
- 网关限流
- Swagger 聚合

### OpenFeign

当前用于服务间同步调用，例如：

- `product-service` 调用 `order-service` 的普通订单内部接口
- `order-service` 调用 `product-service` 的库存回补内部接口
- `product-service` 调用 `auth-service` 地址接口
- `ai-service` 调用商品与秒杀商品知识接口

## 认证与安全

### JWT

当前做法：

- `auth-service` 签发 token
- `gateway` 统一校验 token
- 通过 Redis 维护 token 黑名单和 token version
- 下游服务通过 `X-User-Id` 接收网关透传的身份

### BCrypt

当前做法：

- 注册时加密密码
- 登录时校验密码
- 修改密码时重新编码

## 数据访问与基础设施

### MyBatis

当前做法：

- 以 `Mapper + XML` 为主
- 主要用于 `auth-service`、`product-service`、`order-service`、`seckill-service`、`ai-service`

### MySQL

核心业务表包括：

- `user`
- `user_address`
- `product`
- `cart_item`
- `normal_order`
- `normal_order_item`
- `seckill_product`
- `seckill_order`
- AI 聊天与知识库相关表

### Redis

当前用途分为 5 类：

- 网关限流
- token 黑名单与 token version
- 普通商品与购物车缓存
- 秒杀库存、秒杀结果、MQ 幂等状态
- AI 会话上下文与近期历史缓存

近期已收口的正确性修复：

- 秒杀重复消息库存误扣
- AI 会话上下文缓存 miss 失真
- 秒杀补偿改为 Lua 原子回滚

### RabbitMQ

当前主要用于：

- 秒杀异步建单
- 死信补偿链路

### Nacos

当前用途：

- 服务注册与发现
- 配置中心

当前配置模型：

- `application.yml`：接入 Nacos 和服务基础入口
- `application-local.yml`：本地兜底
- Nacos：共享配置和服务私有配置

## 接口文档与调试

### SpringDoc OpenAPI

当前做法：

- 各服务暴露自己的 `/v3/api-docs`
- `gateway` 聚合 Swagger UI

### Swagger UI

当前入口：

- `http://localhost:8080/swagger-ui.html`

## 前端技术栈

| --- | --- | --- |

- 秒杀页
- 购物车
- AI 助手
- 结算页

- 账户信息
- 密码安全
