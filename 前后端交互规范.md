# 前后端交互规范

> 本文档用于约束 `flash-sale-ui` 与后端微服务的联调口径。  
> 这一版在原有页面访问规则和接口说明之外，补充了所有当前接口涉及的 `DTO / VO / PO` 速查信息。

## 1. 统一访问入口

- 前端开发地址：`http://localhost:5173`
- 网关地址：`http://localhost:8080`
- Swagger 聚合入口：`http://localhost:8080/swagger-ui.html`
- AI 服务 OpenAPI JSON：`http://localhost:8080/v3/api-docs/ai-service`

约束：

- 前端所有业务请求统一走网关
- 前端不直接访问单个微服务端口

## 2. 前端页面访问规则

### 2.1 匿名可访问页面

- `/#/app/home`
- `/#/app/flash`
- `/#/login`
- `/#/register`

### 2.2 登录后可访问页面

- `/#/app/cart`
- `/#/app/profile`
- `/#/app/profile/orders`
- `/#/app/profile/account`
- `/#/app/profile/security`
- `/#/checkout`

### 2.3 路由守卫规则

- 带 `requiresAuth` 的页面需要登录
- 未登录访问受保护页面时，前端跳转登录页并带上 `redirect`
- 已登录访问 `guestOnly` 页面时，前端回到首页

## 3. 鉴权口径

### 3.1 请求头

除白名单接口外，统一带：

```http
Authorization: Bearer <token>
```

### 3.2 下游身份

前端不传 `X-User-Id`。  
该字段由 Gateway 鉴权成功后自动补充。

### 3.3 401 处理

当前前端约定：

- 收到 `401` 后清理本地会话
- 跳转登录页
- 不允许页面停留在伪登录状态

## 4. 统一响应结构

所有业务接口统一返回：

```json
{
  "code": 200,
  "message": "成功",
  "data": {},
  "timestamp": "2026-03-26T12:00:00"
}
```

前端处理建议：

1. 先看 HTTP 状态码
2. 再看 `code`
3. 最后读取 `data`

## 5. 接口与模型总览

说明：

- `DTO`：请求对象或服务间传输对象
- `VO`：接口响应对象
- `PO`：数据库持久化对象
- 有些接口没有单独的请求体 DTO，而是通过 `PathVariable` 或 `RequestParam` 接收参数
- `PO` 不是前端直接使用的结构，但加入本文档是为了让接口和落库结构能一起对照

## 6. 认证与账户接口

### 6.1 `POST /auth/login`

请求 DTO：`LoginRequestDTO`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `username` | `String` | 用户名，长度 3 到 32 |
| `password` | `String` | 密码，长度 6 到 64 |

响应 VO：`UserVO`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `userId` | `Long` | 用户 ID |
| `username` | `String` | 用户名 |
| `token` | `String` | 登录成功后的 JWT |

关联 PO：

- 当前仓库中没有单独暴露 `UserPO` 类
- 登录态主要通过 JWT 返回，不直接返回数据库实体

### 6.2 `POST /auth/register`

请求 DTO：`RegisterRequestDTO`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `username` | `String` | 用户名，长度 3 到 32 |
| `password` | `String` | 密码，长度 6 到 64 |

响应：

- 成功时 `data` 为空

关联 PO：

- 当前仓库中没有单独暴露 `UserPO` 类

### 6.3 `POST /auth/updatePassword`

| `oldPassword` | `String` | 旧密码 |

## 16. 搜索与分类联调口径

当前普通商品搜索统一口径：

- 输入商品名，按商品名称模糊匹配
- 输入分类词，如“酒水饮料”，按分类命中商品
- 输入更短的分类关键词，如“饮料”，也允许命中对应分类
- 左侧显式分类筛选继续使用 `categoryId`

### 17.1 问答请求 DTO：`ChatRequestDTO`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `question` | `String` | 用户问题，必填，最大 500 字符 |
| `productId` | `Long` | 当前商品 ID，商品详情页提问时建议传入 |
| `productName` | `String` | 当前商品名称，已知时建议传入 |
| `productType` | `String` | 商品类型，`normal` 或 `seckill`，用于区分普通商品和秒杀商品 |
| `sessionId` | `String` | 会话 ID，用于多轮上下文 |
| `contextType` | `String` | 上下文来源，例如 `product-detail` 或 `global-assistant` |

说明：

- 商品详情页提问时，建议同时传 `productId`、`productName`、`productType`
- `productType + productId` 共同构成商品身份，避免普通商品和秒杀商品同 ID 时串知识
