# Flash Sale System 前后端交互规范（基于当前代码实现）

> 本文档按**当前仓库代码**整理，目标是让前端可以直接联调。若后端接口后续调整，请同步更新本文档。

---

## 1. 网关与服务访问约定

## 1.1 推荐访问入口（Gateway）
- Gateway 默认端口：`8080`
- 已配置路由前缀：
  - `/auth/**` → auth-service
  - `/seckill/**` → seckill-service
  - `/order/**` → order-service

## 1.2 当前路由注意事项
- `SeckillProductController` 的接口路径是 `/seckill-product/**`，**当前 Gateway 已配置该前缀路由**，前端可统一走 `8080` 网关访问。
- 由于 `/seckill-product/**` 不在白名单内，当前通过 Gateway 访问商品接口时也需要携带登录 token。
- `SeckillOrderController` 路径是 `/seckill-order/**`，当前 Gateway 也未配置该前缀；前端若不用该组接口可忽略（`/order/**` 已可覆盖订单查询）。

---

## 2. 鉴权与请求头规范

## 2.1 登录白名单
- 白名单：`/auth/login`、`/auth/register`
- 其他走 Gateway 的接口默认需要登录。

## 2.2 Token 传递
- 头部：`Authorization: Bearer <token>`
- Gateway 鉴权通过后会向下游附加：`X-User-Id: <userId>`
- 秒杀相关接口（如 `/seckill/{productId}`、`/seckill/result/{productId}`）依赖 `X-User-Id`。

## 2.3 未登录/Token 无效响应
- Gateway 会直接返回 **HTTP 401**（一般无统一 JSON 包体）。
- 前端建议统一拦截 401，跳转登录页并清理本地 token。

---

## 3. 统一返回结构（业务层）

后端业务接口统一使用 `Result<T>`：

```json
{
  "code": 200,
  "message": "success",
  "data": {},
  "timestamp": "2026-03-16T15:30:00"
}
```

字段说明：
- `code`：业务码（不是 HTTP 状态码）
- `message`：业务提示
- `data`：业务数据（可为对象/数组/null）
- `timestamp`：返回时间

> 当前代码已补齐 `Result.error(...)` 的 `timestamp`，成功和业务失败返回格式保持一致。

常见业务码：
- `200`：success
- `400`：param error
- `401`：unauthorized
- `403`：forbidden
- `500`：server error
- `2001`：stock empty
- `2002`：repeat seckill
- `2003`：business error

> 说明：当前秒杀接口很多失败场景也会返回 `code=200`，失败语义放在 `data` 里的状态字段中（见第 5 节）。

## 4. 认证接口（Auth）

基准路径：`/auth`

### 4.1 登录
- `POST /auth/login`
- 请求体：
```json
{
  "username": "alice",
  "password": "123456"
}
```
- 成功 `data`：
```json
{
  "userId": 1,
  "username": "alice",
  "token": "eyJ..."
}
```
- 前端处理：保存 token；后续请求统一带 `Authorization: Bearer <token>`。

### 4.2 注册
- `POST /auth/register`
- 请求体同登录
- 成功返回：`Result.success()`（`data` 通常为 `null`）
- 若用户名已存在：返回 `code=2003`，`message=用户名已存在`

### 4.3 登出（占位）
- `POST /auth/logout`
- 当前实现固定返回成功，前端可本地清 token 即可。

### 4.4 当前用户信息（占位）
- `POST /auth/me`
- 当前实现返回成功但 `data` 为空，建议前端暂不依赖。

## 5. 秒杀与商品接口（Seckill）

## 5.1 普通商品列表
- `GET /product/products`（当前推荐直接走 Gateway：`http://localhost:8080/product/products`）
- Query 参数（可选）：
  - `name`：商品名模糊查询
  - `status`：0 下架 / 1 上架
  - `categoryId`：分类 ID
- 返回 `data`：`ProductVO[]`
  - `id, name, subtitle, categoryId, price, marketPrice, stock, status, mainImage, detail, createTime, updateTime`

## 5.2 普通商品详情
- `GET /product/products/{id}`
- 返回 `data`：`ProductVO`

## 5.3 秒杀商品列表
- `GET /seckill-product/products`（当前推荐直接走 Gateway：`http://localhost:8080/seckill-product/products`）
- Query 参数（可选）：
  - `name`：商品名模糊查询
  - `status`：0 下架 / 1 上架
- 返回 `data`：`SeckillProductVO[]`
  - `id, name, price, seckillPrice, stock, status, startTime, endTime`

## 5.4 秒杀商品详情
- `GET /seckill-product/products/{id}`
- 返回 `data`：`SeckillProductVO`

## 5.5 发起秒杀
- `POST /seckill/{productId}`（走 Gateway）
- 需要登录（依赖网关注入 `X-User-Id`）
- 返回 `data`：`SeckillResultVO`
  - `success: true/false`
  - `message: 文案`
  - `productId`
  - `orderId`（异步下单阶段通常为空）

示例（进入排队）：
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "success": true,
    "message": "秒杀成功，订单处理中",
    "productId": 1,
    "orderId": null
  },
  "timestamp": "2026-03-16T15:30:00"
}
```

## 5.6 查询秒杀结果（轮询）
- `GET /seckill/result/{productId}`
- 返回 `data`：`SeckillStatusVO`
  - `status = 1`：秒杀成功（`orderId` 有值）
  - `status = 0`：排队中
  - `status = -1`：秒杀失败
  - `message`：对应文案

示例（成功）：
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "status": 1,
    "message": "秒杀成功",
    "orderId": 10001
  },
  "timestamp": "2026-03-16T15:30:02"
}
```

## 6. 订单接口（Order）

基准路径：`/order`（走 Gateway）

### 6.1 订单列表
- `GET /order/orders`
- 当前实现只返回 **当前登录用户** 的订单，不再返回全量订单
- 返回 `data`：`SeckillOrderVO[]`
  - `id, userId, productId, seckillPrice, status, createTime`

### 6.2 订单详情
- `GET /order/orderDetail/{id}`
- 当前实现会按 `X-User-Id` 校验归属，不存在或非本人订单时返回 `code=2003`、`message=订单不存在`
- 返回 `data`：`SeckillOrderVO`

> 备注：仓库里还有一组 `/seckill-order/**` 的重复订单接口，功能相似，前端建议统一用 `/order/**`。

## 7. 前端推荐联调流程（秒杀核心）

1. 登录：`POST /auth/login`，保存 `token`。
2. 拉普通商品：`GET /product/products`。
3. 拉秒杀商品：`GET /seckill-product/products`。
4. 点击秒杀：`POST /seckill/{productId}`。
5. 若返回 `data.success=true`，开始轮询：`GET /seckill/result/{productId}`（建议 300~500ms 一次，最长 10~15s）。
6. 按 `data.status` 处理：
   - `1`：跳订单详情页（可用 `orderId` 查询）
   - `0`：继续轮询
   - `-1`：展示失败提示并停止轮询

---

## 8. 前端实现建议（当前版本）

- Axios 拦截器：
  - 请求拦截器统一注入 `Authorization`。
  - 响应拦截器统一处理 HTTP 401。
- 业务层判断优先级：
  1) 先看 HTTP 状态（401/5xx）
  2) 再看 `Result.code`
  3) 对秒杀结果再看 `data.success` 或 `data.status`
- 秒杀按钮防抖：提交后立即禁用，直到轮询结束或失败。
- 倒计时展示：基于商品 `startTime/endTime` 控制“未开始/进行中/已结束”状态。
