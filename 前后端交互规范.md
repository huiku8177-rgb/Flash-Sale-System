# 前后端交互规范

> 本文档面向 `flash-sale-ui` 与后端服务联调，以当前仓库实现为准。

## 1. 统一访问入口

- 网关地址：`http://localhost:8080`
- 前端开发地址：`http://localhost:5173`
- 前端所有业务请求统一走网关，不直连微服务

当前网关路由：
- `/auth/**`
- `/product/**`
- `/seckill/**`
- `/seckill-product/**`
- `/order/**`

## 2. 页面访问规则

### 匿名可访问页面
- `/#/app/home`
- `/#/app/flash`

### 登录后可访问页面
- `/#/app/cart`
- `/#/app/profile`
- `/#/checkout`

### 特殊交互规则
- 未登录点击商品“查看详情”会直接跳转登录页
- 未登录访问购物车、个人中心、结算页会被前端路由守卫拦截
- 购物车与结算页是两张独立页面，不共用同一个页面容器

## 3. 鉴权规则

### 3.1 白名单接口
- `POST /auth/login`
- `POST /auth/register`
- `GET /product/products`
- `GET /product/products/**`
- `GET /seckill-product/products`
- `GET /seckill-product/products/**`
- `/swagger-ui.html`
- `/swagger-ui/**`
- `/v3/api-docs`
- `/v3/api-docs/**`
- `/error`
- `/favicon.ico`

### 3.2 受保护接口
除白名单外，其余接口统一要求：

```http
Authorization: Bearer <token>
```

前端保存登录态后，请求头应自动携带：

```http
Authorization: Bearer <token>
```

网关鉴权成功后，会向下游服务补充：

```http
X-User-Id: <userId>
```

前端不需要手动拼装 `X-User-Id`。

### 3.3 401 处理
- 后端返回 `401` 时，前端统一清理本地登录态
- 然后跳转登录页
- 不允许页面静默留在当前状态

## 4. 统一响应结构

所有业务接口统一返回：

```json
{
  "code": 200,
  "message": "成功",
  "data": {},
  "timestamp": "2026-03-22T12:00:00"
}
```

字段含义：
- `code`：业务码
- `message`：中文提示
- `data`：业务数据
- `timestamp`：响应时间

前端处理建议：
- 先判断 HTTP 状态
- 再判断 `code`
- 再读取 `data`

## 5. 认证接口

基础路径：`/auth`

### 5.1 登录
- `POST /auth/login`

请求体：

```json
{
  "username": "neo",
  "password": "123456"
}
```

成功返回 `data`：

```json
{
  "userId": 10001,
  "username": "neo",
  "token": "eyJ..."
}
```

### 5.2 注册
- `POST /auth/register`

请求体：

```json
{
  "username": "neo",
  "password": "123456"
}
```

### 5.3 当前用户
- `GET /auth/me`

### 5.4 修改密码
- `POST /auth/updatePassword`

请求体：

```json
{
  "oldPassword": "123456",
  "newPassword": "654321"
}
```

### 5.5 退出登录
- `POST /auth/logout`

说明：
- 当前前端仍以“清理本地 token”为主
- 后端该接口更多用于统一语义

## 6. 普通商品接口

基础路径：`/product`

### 6.1 商品列表
- `GET /product/products`

可选查询参数：
- `name`
- `status`
- `categoryId`

说明：
- 当前为公开接口
- 未登录也可以浏览首页普通商品

### 6.2 商品详情
- `GET /product/products/{id}`

说明：
- 后端接口本身为公开接口
- 但前端页面交互层已要求“查看详情前先登录”

## 7. 购物车接口

基础路径：`/product/cart`

### 7.1 查询购物车
- `GET /product/cart/items`

返回字段建议重点使用：
- `id`
- `productId`
- `productName`
- `productSubtitle`
- `productImage`
- `price`
- `marketPrice`
- `stock`
- `status`
- `quantity`
- `selected`
- `itemAmount`
- `canCheckout`

### 7.2 加入购物车
- `POST /product/cart/items`

请求体：

```json
{
  "productId": 1001,
  "quantity": 1,
  "selected": true
}
```

### 7.3 更新购物车项
- `PUT /product/cart/items/{id}`

请求体：

```json
{
  "quantity": 2,
  "selected": true
}
```

### 7.4 删除单个购物车项
- `DELETE /product/cart/items/{id}`

### 7.5 清空购物车
- `DELETE /product/cart/items`

可选参数：
- `selectedOnly=true`

说明：
- `selectedOnly=true` 表示只清空已选商品
- 不传时视为清空全部购物车项

## 8. 普通结算与普通订单接口

### 8.1 当前结算规则
普通下单不再由前端直接提交商品 `items`。

当前正确流程是：
1. 前端在购物车中勾选商品，或点击“立即购买”
2. 前端进入独立结算页 `/#/checkout`
3. 结算页填写收货信息和备注
4. 前端调用 `POST /product/normal-orders`
5. 后端读取当前用户已选购物车商品完成建单
6. 前端调用普通订单模拟支付接口
7. 支付成功后返回购物车页并自动刷新

### 8.2 创建普通订单
- `POST /product/normal-orders`

请求体：

```json
{
  "remark": "工作日白天送达",
  "addressSnapshot": "{\"receiver\":\"Neo\",\"mobile\":\"13800000000\",\"detail\":\"上海市浦东新区xxx路100号\"}"
}
```

说明：
- 当前请求体不再传 `items`
- 商品来源以后端已选购物车项为准
- 收货人、手机号、收货地址未填写完整时，前端不允许发起支付流程

### 8.3 查询普通订单列表
- `GET /order/normal-orders`

可选参数：
- `status`

### 8.4 查询普通订单详情
- `GET /order/normal-orders/{id}`

### 8.5 模拟支付普通订单
- `POST /order/normal-orders/{id}/pay`

### 8.6 查询普通订单支付状态
- `GET /order/normal-orders/{id}/pay-status`

## 9. 秒杀接口

### 9.1 秒杀商品列表
- `GET /seckill-product/products`

### 9.2 秒杀商品详情
- `GET /seckill-product/products/{id}`

### 9.3 发起秒杀
- `POST /seckill/{productId}`

说明：
- 该接口需要登录
- 前端不能只看 `Result.code`
- 还要结合返回 `data.success` 或结果接口继续判断

### 9.4 查询秒杀结果
- `GET /seckill/result/{productId}`

状态含义：
- `1`：抢购成功
- `0`：排队中
- `-1`：抢购失败

前端建议：
- 以 `300ms ~ 500ms` 轮询
- 设置最大轮询时长，避免无限等待

## 10. 秒杀订单接口

基础路径：`/order`

### 10.1 秒杀订单列表
- `GET /order/seckill-orders`

### 10.2 秒杀订单详情
- `GET /order/seckill-orders/{id}`

### 10.3 模拟支付秒杀订单
- `POST /order/seckill-orders/{id}/pay`

### 10.4 查询秒杀订单支付状态
- `GET /order/seckill-orders/{id}/pay-status`

## 11. 前端实现约定

### 11.1 路由
- 使用 `createWebHashHistory()`
- 页面地址格式为 `http://localhost:5173/#/xxx`

### 11.2 登录态
- token 保存在本地存储
- 路由守卫按 `requiresAuth` 控制访问
- 商品详情点击前额外做一次登录校验

### 11.3 购物车与结算
- 购物车已切到后端接口，不再使用本地 `localStorage` 作为主数据源
- 结算页是独立页面
- “立即购买”会将当前商品作为本次结算目标后跳转结算页
- 支付完成后返回购物车页并自动刷新

## 12. 联调建议

推荐按以下顺序联调：
1. 登录 / 注册
2. 普通商品列表 / 详情
3. 购物车增删改查
4. 立即购买与独立结算页
5. 普通订单创建、模拟支付、支付状态
6. 秒杀下单与轮询
7. 秒杀订单模拟支付
