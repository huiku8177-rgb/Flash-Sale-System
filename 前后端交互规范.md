# 前后端交互规范

## 文档定位

本文档描述当前前端页面访问规则、前端实际 API 调用口径，以及对外接口的统一约定。

## 统一访问入口

标准联调入口：

- 网关：`http://localhost:8080`

前端当前客户端：

- `gatewayClient`：默认走网关
- `productClient`：默认也走网关，可通过环境变量改为直连商品相关服务，仅建议本地调试使用

## 前端页面访问规则

### 匿名可访问页面

- `/#/login`
- `/#/register`
- `/#/app/home`
- `/#/app/flash`

### 登录后可访问页面

- `/#/checkout`
- `/#/app/cart`
- `/#/app/assistant`
- `/#/app/profile`
- `/#/app/profile/orders`
- `/#/app/profile/account`
- `/#/app/profile/security`

### 路由守卫规则

- `requiresAuth = true` 时，未登录跳转到登录页
- `guestOnly = true` 时，已登录跳转回首页
- 登录页保留 `redirect` 参数，便于登录后回跳

## 鉴权口径

### 请求头

标准请求头：

- `Authorization: Bearer <token>`

前端当前会附带 `X-User-Id`，但对标准联调来说：

- 外部传入的 `X-User-Id` 不可信
- 网关会移除并改写为真实用户身份

### 401 处理

当前前端行为：

- 清空本地会话
- 跳转到登录页
- 自动带上当前页面 `redirect`

## 统一响应结构

所有外部接口统一遵循：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

前端约定：

- `code !== 200` 按业务错误处理
- `401` 统一走登录失效逻辑
- 业务数据从 `data` 读取

## 页面到接口的主要映射

### 登录 / 注册

- 登录页调用 `/auth/login`
- 注册页调用 `/auth/register`

### 首页

- 商品区调用 `/product/products`
- 秒杀区调用 `/seckill-product/products`

### 商品详情与秒杀详情

- 普通商品详情调用 `/product/products/{id}`
- 秒杀商品详情调用 `/seckill-product/products/{id}`

### 购物车与结算

- 购物车页调用 `/product/cart/items`
- 结算页创建普通订单调用 `/product/normal-orders`

### 订单中心

- 普通订单走 `/order/normal-orders/**`
- 秒杀订单走 `/order/seckill-orders/**`

### AI 助手

- 问答走 `/ai/chat`
- 会话列表与详情走 `/ai/chat/sessions/**`

## 联调注意事项

### 1. 前端默认应走网关

除本地调试外，不建议让前端直接访问单体服务地址。

### 2. 外部 `X-User-Id` 不作为真实身份依据

标准链路中，真实用户身份以网关写入的头为准。

### 3. 普通订单创建与普通订单查询分属两个服务

这是当前架构有意设计，不是接口遗漏：

- 创建入口在 `product-service`
- 生命周期管理在 `order-service`

### 4. 秒杀状态与秒杀订单不是同一层接口

- 秒杀结果轮询走 `/seckill/result/{productId}`
- 已落库后的订单查询走 `/order/seckill-orders/**`
