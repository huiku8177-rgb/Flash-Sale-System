# 前后端交互规范

> 本文档以当前仓库代码为准，面向 `flash-sale-ui` 与后端服务联调。

## 1. 访问入口

- 网关地址：`http://localhost:8080`
- 前端默认统一走网关，不再直连单体服务

当前网关已配置：

- `/auth/**`
- `/order/**`
- `/seckill/**`
- `/seckill-product/**`
- `/product/**`

## 2. 鉴权规则

- 白名单：`POST /auth/login`、`POST /auth/register`
- 其他接口都需要 `Authorization: Bearer <token>`
- Gateway 鉴权成功后会向下游透传 `X-User-Id`
- 商品列表和商品详情当前也属于受保护接口
- 前端统一拦截 HTTP `401`，清理本地登录态并跳转登录页

## 3. 统一返回结构

所有业务接口统一使用 `Result<T>`：

```json
{
  "code": 200,
  "message": "success",
  "data": {},
  "timestamp": "2026-03-19T12:00:00"
}
```

字段说明：

- `code`：业务码
- `message`：业务提示
- `data`：业务数据，可能是对象、数组或 `null`
- `timestamp`：服务端响应时间

常见业务码：

- `200`：成功
- `400`：参数错误
- `401`：未认证
- `403`：无权限
- `500`：系统异常
- `2001`：库存不足
- `2002`：重复秒杀
- `2003`：业务异常

## 4. Auth 接口

基础路径：`/auth`

### 4.1 登录

- `POST /auth/login`

请求体：

```json
{
  "username": "neo",
  "password": "123456"
}
```

成功返回 `data`：

```json
{
  "userId": 1,
  "username": "neo",
  "token": "eyJ..."
}
```

### 4.2 注册

- `POST /auth/register`

请求体：

```json
{
  "username": "neo",
  "password": "123456"
}
```

成功返回：

- `data = null`

### 4.3 退出登录

- `POST /auth/logout`

说明：

- 当前是轻量占位实现
- 前端仍以“本地清 token”为主

### 4.4 当前用户

- `GET /auth/me`

说明：

- 依赖 Gateway 注入的 `X-User-Id`
- 当前返回的 `UserVO` 字段为：`userId`、`username`、`token`
- `token` 在该接口下一般可为空

### 4.5 修改密码

- `POST /auth/updatePassword`

请求体：

```json
{
  "oldPassword": "123456",
  "newPassword": "654321"
}
```

当前校验规则：

- 旧密码不能为空
- 新密码不能为空
- 旧密码必须正确
- 新密码不能与旧密码相同

## 5. 普通商品接口

基础路径：`/product`

### 5.1 商品列表

- `GET /product/products`

可选查询参数：

- `name`
- `status`
- `categoryId`

当前返回字段：

- `id`
- `name`
- `subtitle`
- `categoryId`
- `price`
- `marketPrice`
- `stock`
- `status`
- `mainImage`
- `detail`
- `createTime`
- `updateTime`

说明：

- 当前默认只查上架商品
- 不存在的商品详情会返回业务错误

### 5.2 商品详情

- `GET /product/products/{id}`

## 6. 秒杀商品与秒杀接口

### 6.1 秒杀商品列表

- `GET /seckill-product/products`

可选查询参数：

- `name`
- `status`

返回字段：

- `id`
- `name`
- `price`
- `seckillPrice`
- `stock`
- `status`
- `startTime`
- `endTime`

### 6.2 秒杀商品详情

- `GET /seckill-product/products/{id}`

### 6.3 发起秒杀

- `POST /seckill/{productId}`

成功进入队列时，`data` 示例：

```json
{
  "success": true,
  "message": "秒杀成功，订单处理中",
  "productId": 1,
  "orderId": null
}
```

说明：

- 前端不能只看 `Result.code`
- 还要继续根据 `data.success` 判断是否进入秒杀流程

### 6.4 查询秒杀结果

- `GET /seckill/result/{productId}`

返回字段：

- `status`
- `message`
- `orderId`

状态含义：

- `1`：秒杀成功
- `0`：排队中
- `-1`：秒杀失败

说明：

- Redis 结果过期后，后端会回查数据库订单兜底
- 前端建议以 `300ms ~ 500ms` 的间隔轮询，最长 `10s ~ 15s`

## 7. 秒杀订单接口

基础路径：`/order`

### 7.1 秒杀订单列表

- `GET /order/orders`

返回字段：

- `id`
- `userId`
- `productId`
- `seckillPrice`
- `status`
- `createTime`

状态含义：

- `0`：待支付
- `1`：已支付
- `2`：已取消

### 7.2 秒杀订单详情

- `GET /order/orderDetail/{id}`

### 7.3 秒杀订单模拟支付

- `POST /order/seckill-orders/{id}/pay`

### 7.4 秒杀订单支付状态

- `GET /order/seckill-pay-status/{id}`

返回字段：

- `orderId`
- `productId`
- `status`
- `paid`
- `seckillPrice`
- `message`

## 8. 普通订单接口

基础路径：`/order`

### 8.1 创建普通订单

- `POST /order/checkout`

请求体：

```json
{
  "items": [
    {
      "productId": 1,
      "quantity": 2
    }
  ],
  "remark": "工作日白天送达",
  "addressSnapshot": "{\"receiver\":\"Neo\",\"mobile\":\"13800000000\",\"detail\":\"上海市浦东新区xxx路100号\"}"
}
```

说明：

- `addressSnapshot` 当前约定传 JSON 字符串
- 后端会校验 JSON 合法性
- 重复商品会合并数量

### 8.2 普通订单列表

- `GET /order/normal-orders`

可选参数：

- `status`

### 8.3 普通订单详情

- `GET /order/normal-orders/{id}`

返回字段：

- `id`
- `orderNo`
- `userId`
- `orderStatus`
- `totalAmount`
- `payAmount`
- `payTime`
- `remark`
- `addressSnapshot`
- `createTime`
- `items`

其中 `items` 内字段包括：

- `id`
- `orderId`
- `userId`
- `productId`
- `productName`
- `productSubtitle`
- `productImage`
- `salePrice`
- `quantity`
- `itemAmount`
- `createTime`

### 8.4 普通订单模拟支付

- `POST /order/normal-orders/{id}/pay`

### 8.5 普通订单支付状态

- `GET /order/pay-status/{id}`

返回字段：

- `orderId`
- `orderNo`
- `orderStatus`
- `paid`
- `payAmount`
- `payTime`
- `message`

普通订单状态含义：

- `0`：待支付
- `1`：已支付
- `2`：已取消
- `3`：已发货
- `4`：已完成

## 9. 前端当前实现约定

- 前端不再使用旧的 `/pay-product/**`
- 前端统一不使用 `/seckill-order/**` 这组历史重复接口
- 购物车当前是前端本地状态，不走后端购物车接口
- 秒杀商品加入购物车只作为“草稿提醒”，正式下单仍需在秒杀页发起秒杀
- 个人中心会将普通订单和秒杀订单在前端聚合展示

## 10. 推荐联调顺序

1. `POST /auth/login`
2. `GET /product/products`
3. `GET /seckill-product/products`
4. `POST /seckill/{productId}`
5. `GET /seckill/result/{productId}`
6. `GET /order/orderDetail/{id}`
7. `POST /order/seckill-orders/{id}/pay`
8. `POST /order/checkout`
9. `GET /order/normal-orders`
10. `POST /order/normal-orders/{id}/pay`
