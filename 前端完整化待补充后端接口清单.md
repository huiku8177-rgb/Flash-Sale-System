# 前端完整化待补充后端接口清单

> 本文档只记录“下一阶段还值得补”的能力，不再重复已经完成的接口。

## 1. 当前前端已经能做什么

`flash-sale-ui` 现在已经接通：

- 登录、注册、退出登录
- 获取当前用户
- 修改密码
- 普通商品列表、详情、分类筛选
- 秒杀商品列表、详情
- 发起秒杀、轮询秒杀结果
- 普通订单创建、详情、模拟支付、支付状态
- 秒杀订单详情、模拟支付、支付状态
- 普通订单与秒杀订单统一展示

所以这份清单不再把 `/auth/me`、`/order/checkout`、`/order/pay-status/{id}` 之类接口列为“待补”。

## 2. 当前最值得补的后端能力

### 2.1 真实购物车服务

当前购物车仍是前端本地状态。若要支持跨端同步、刷新不丢失、后端统一结算，建议补：

- `GET /cart/items`
- `POST /cart/items`
- `PUT /cart/items/{itemId}`
- `DELETE /cart/items/{itemId}`
- `DELETE /cart/items`

建议返回字段：

- `itemId`
- `productId`
- `productType`
- `productName`
- `productImage`
- `price`
- `seckillPrice`
- `quantity`
- `stock`
- `selected`

### 2.2 用户资料扩展

当前 `/auth/me` 已能返回基础用户，但字段较少。若要把个人中心做完整，建议继续补：

- `nickname`
- `avatar`
- `mobile`
- `email`

如果后续要支持编辑资料，再增加：

- `PUT /auth/me`

### 2.3 地址管理

普通订单当前通过 `addressSnapshot` 直接落快照，能跑通，但不适合长期使用。建议补：

- `GET /user/addresses`
- `POST /user/addresses`
- `PUT /user/addresses/{id}`
- `DELETE /user/addresses/{id}`

这样前端可以在结算页选择地址，而不是每次手填。

### 2.4 统一订单聚合接口

当前“我的订单”是在前端把普通订单和秒杀订单拼起来。若想降低前端聚合成本，建议补一个统一接口：

- `GET /order/all-orders`

可选参数：

- `pageNum`
- `pageSize`
- `status`
- `orderType`

建议统一返回字段：

- `id`
- `orderNo`
- `orderType`
- `status`
- `amount`
- `summary`
- `createTime`

### 2.5 首页装修与运营位

当前首页轮播、快捷区、热门搜索还是前端静态配置。若要更像真实商城，建议补：

- `GET /home/config`
- `GET /home/channels`
- `GET /search/hot-keywords`
- `GET /search/suggest?keyword=...`

## 3. 秒杀方向仍可增强的接口

### 3.1 秒杀场次

当前秒杀页通过商品时间字段自行判断状态，尚未引入“活动场次”概念。建议补：

- `GET /seckill/activities`
- `GET /seckill/activities/{activityId}/products`

### 3.2 秒杀结果可视化

当前轮询结果已经能闭环，但返回值还比较轻。若后续要做更强的排队感知，建议补：

- `queuePosition`
- `estimatedWaitMs`
- `finalStatus`

## 4. 商品能力仍可增强的方向

### 4.1 分类接口

现在首页分类来自商品列表聚合。若后端提供独立分类接口，会更稳定：

- `GET /product/categories`

### 4.2 搜索与推荐

当前搜索仍基于商品列表查询参数。若后续加强商城感，可以补：

- `GET /product/search?keyword=...&pageNum=1&pageSize=20`
- `GET /product/recommend`

## 5. 优先级建议

如果你准备继续补后端接口，我建议按这个顺序：

1. 真实购物车服务
2. 地址管理
3. 统一订单聚合接口
4. 用户资料扩展
5. 首页装修与热词
6. 秒杀场次接口

## 6. 当前不需要再补的接口

这些能力已经完成，不需要再作为待办写进文档：

- `GET /auth/me`
- `POST /auth/updatePassword`
- `POST /order/checkout`
- `GET /order/normal-orders`
- `GET /order/normal-orders/{id}`
- `POST /order/normal-orders/{id}/pay`
- `GET /order/pay-status/{id}`
- `POST /order/seckill-orders/{id}/pay`
- `GET /order/seckill-pay-status/{id}`
