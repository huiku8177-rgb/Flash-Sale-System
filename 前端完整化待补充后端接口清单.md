# 前端完整化待补充后端接口清单

> 本文档只记录“当前仍值得继续补”的后端能力，不再重复已经完成并联调通过的接口。

## 1. 已经完成的主链路

当前前端已经可以正常联调以下能力：

- 登录、注册、退出登录
- 获取当前用户信息
- 修改密码
- 普通商品列表、详情
- 秒杀商品列表、详情
- 购物车增删改查
- 商品卡片“立即购买”
- 独立结算页
- 基于已选购物车商品创建普通订单
- 普通订单列表、详情、模拟支付、支付状态查询
- 发起秒杀、轮询秒杀结果
- 秒杀订单列表、详情、模拟支付、支付状态查询

说明：
- 普通下单入口已经统一为 `POST /product/normal-orders`
- 普通商品链路已经是 `product-service -> OpenFeign -> order-service`
- 购物车已经从前端本地状态切换到后端接口

## 2. 当前最值得继续补的后端能力

## 2.1 地址管理

当前问题：
- 结算页仍然让用户手填 `addressSnapshot`
- 能跑通，但还不够像正式商城

建议新增接口：
- `GET /user/addresses`
- `POST /user/addresses`
- `PUT /user/addresses/{id}`
- `DELETE /user/addresses/{id}`

建议返回字段：
- `id`
- `receiver`
- `mobile`
- `province`
- `city`
- `district`
- `detail`
- `defaultAddress`

收益：
- 结算页可直接选择地址
- 支持默认地址回填
- 可以逐步摆脱手填快照模式

## 2.2 购物车结算预览

当前问题：
- 结算页目前更多依赖前端已有状态展示
- 缺少一个后端统一返回“本次可结算数据”的接口

建议新增接口：
- `GET /product/cart/preview`
或
- `POST /product/cart/preview`

建议返回内容：
- 已选中的可结算商品
- 失效商品
- 库存不足商品
- 合计金额
- 可结算件数

收益：
- 结算页口径更以后端为准
- 可以把金额、库存、可支付状态统一校准
- 后续更适合扩展优惠、运费、地址校验

## 2.3 批量选中 / 取消选中

当前问题：
- 购物车单项更新已经有了
- 但“全选 / 取消全选”仍适合由后端提供批量能力

建议新增接口：
- `PUT /product/cart/items/selection`

建议请求体：

```json
{
  "selected": true
}
```

收益：
- 前端全选交互更简单
- 不需要逐项调用更新接口

## 2.4 普通订单取消 / 超时关闭

当前问题：
- 现在普通订单支持创建和模拟支付
- 但缺少“未支付取消”与订单生命周期补全

建议新增接口：
- `POST /order/normal-orders/{id}/cancel`

后续可扩展：
- 定时关闭超时未支付订单

收益：
- 订单流程更完整
- 方便演示真实商城订单状态流转

## 2.5 统一订单聚合接口

当前问题：
- 个人中心目前仍然要分别查询普通订单和秒杀订单后再前端聚合

建议新增接口：
- `GET /order/all-orders`

可选参数：
- `status`
- `orderType`
- `pageNum`
- `pageSize`

收益：
- 个人中心接口更统一
- 前端数据组装更轻

## 2.6 首页运营接口

当前问题：
- 首页轮播、频道、热词仍偏前端静态配置

建议新增接口：
- `GET /home/config`
- `GET /home/channels`
- `GET /search/hot-keywords`

收益：
- 首页更像真实商城
- 后续便于扩展推荐位和运营配置

## 3. 推荐实施顺序

我建议按下面顺序继续推进：

1. 地址管理
2. 购物车结算预览
3. 批量选中 / 取消选中
4. 普通订单取消 / 超时关闭
5. 统一订单聚合接口
6. 首页运营接口
