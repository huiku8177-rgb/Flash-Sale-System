# 前端完整化待补充后端接口清单

> 本文档只记录“下一阶段仍然值得补”的能力，不再重复已经完成并上线的接口。

## 1. 当前前端已经打通的能力

`flash-sale-ui` 目前已经能够正常联通以下后端能力：

- 登录、注册、退出登录
- 获取当前用户信息
- 修改密码
- 普通商品列表、详情、分类筛选
- 秒杀商品列表、详情
- 发起秒杀、轮询秒杀结果
- 普通订单创建、详情、模拟支付、支付状态查询
- 秒杀订单列表、详情、模拟支付、支付状态查询
- 普通订单与秒杀订单统一展示

说明：

- 普通商品下单入口已经迁移为 `POST /product/normal-orders`
- 普通商品链路已经拆分为 `product-service -> OpenFeign -> order-service`
- 秒杀订单标准接口已经统一为 `/order/seckill-orders/**`

所以这份清单不再把 `/auth/me`、`/product/normal-orders`、`/order/normal-orders/{id}/pay-status` 之类已完成接口列为“待补”。

## 2. 当前最值得补的后端能力

### 2.1 真实购物车服务

当前购物车仍然主要依赖前端本地状态。若要支持跨端同步、刷新不丢失、统一结算，建议补充：

- `GET /cart/items`
- `POST /cart/items`
- `PUT /cart/items/{itemId}`
- `DELETE /cart/items/{itemId}`
- `DELETE /cart/items`

建议返回字段：

- `itemId`
- `productId`
- `productType`
- `productName`
- `productImage`
- `price`
- `seckillPrice`
- `quantity`
- `stock`
- `selected`

### 2.2 用户资料扩展

当前 `/auth/me` 已能返回基础用户信息，但字段仍然偏少。若要把个人中心做完整，建议继续补充：

- `nickname`
- `avatar`
- `mobile`
- `email`

如果后续要支持编辑资料，可以再增加：

- `PUT /auth/me`

### 2.3 地址管理

普通订单当前通过 `addressSnapshot` 直接落快照，能够跑通流程，但不适合作为长期方案。建议补充：

- `GET /user/addresses`
- `POST /user/addresses`
- `PUT /user/addresses/{id}`
- `DELETE /user/addresses/{id}`

这样前端可以在结算页直接选择地址，而不是每次手工填写。

### 2.4 统一订单聚合接口

当前“我的订单”仍然由前端分别查询普通订单和秒杀订单后再聚合。如果想降低前端组合成本，建议新增：

- `GET /order/all-orders`

可选参数：

- `pageNum`
- `pageSize`
- `status`
- `orderType`

建议统一返回字段：

- `id`
- `orderNo`
- `orderType`
- `status`
- `amount`
- `summary`
- `createTime`

### 2.5 首页装修与运营位

当前首页轮播、快捷区、热门搜索仍以前端静态配置为主。若希望更接近真实商城，建议补充：

- `GET /home/config`
- `GET /home/channels`
- `GET /search/hot-keywords`
- `GET /search/suggest?keyword=...`

## 3. 秒杀方向仍可增强的接口

### 3.1 秒杀场次

当前秒杀页主要依赖商品时间字段判断状态，还没有引入“活动场次”概念。建议补充：

- `GET /seckill/activities`
- `GET /seckill/activities/{activityId}/products`

### 3.2 秒杀结果可视化

当前轮询结果已经能够闭环，但返回值仍然较轻。若后续要加强排队感知，建议新增字段：

- `queuePosition`
- `estimatedWaitMs`
- `finalStatus`

## 4. 商品能力仍可增强的方向

### 4.1 分类接口

当前首页分类来自商品列表结果聚合。如果后端提供独立分类接口，前端会更稳定：

- `GET /product/categories`

### 4.2 搜索与推荐

当前搜索仍然依赖商品列表查询参数。如果后续要加强商城体验，可以补充：

- `GET /product/search?keyword=...&pageNum=1&pageSize=20`
- `GET /product/recommend`

## 5. 优先级建议

如果准备继续完善后端接口，我建议按下面顺序推进：

1. 真实购物车服务
2. 地址管理
3. 统一订单聚合接口
4. 用户资料扩展
5. 首页装修与热搜
6. 秒杀场次接口

## 6. 当前不需要再补的接口

下面这些能力已经完成，不需要再作为待办写入文档：

- `GET /auth/me`
- `POST /auth/updatePassword`
- `POST /product/normal-orders`
- `GET /order/normal-orders`
- `GET /order/normal-orders/{id}`
- `POST /order/normal-orders/{id}/pay`
- `GET /order/normal-orders/{id}/pay-status`
- `GET /order/seckill-orders`
- `GET /order/seckill-orders/{id}`
- `POST /order/seckill-orders/{id}/pay`
- `GET /order/seckill-orders/{id}/pay-status`
