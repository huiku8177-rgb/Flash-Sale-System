# 前端完整化待补充后端接口清单

本文针对当前前端页面结构整理，说明为了把“桌面电商首页 / 闪购会场 / 购物车 / 个人中心”做完整，后端还需要补哪些接口。

---

## 1. 当前已经够用的接口

你现在已有的接口，已经能支撑当前这版前端基础功能：

- `POST /auth/login`
- `POST /auth/register`
- `POST /auth/logout`（目前是占位实现）
- `GET /product/products`
- `GET /product/products/{id}`
- `GET /seckill-product/products`
- `GET /seckill-product/products/{id}`
- `POST /seckill/{productId}`
- `GET /seckill/result/{productId}`
- `GET /order/orders`
- `GET /order/orderDetail/{id}`

这意味着当前已经能完成：

- 登录 / 注册
- 首页普通商品展示
- 普通商品详情查看
- 秒杀页发起抢购
- 轮询秒杀结果
- 我的订单查询

---

## 2. 首页还建议补的接口

当前首页已经改成更接近淘宝网页版的结构：左侧类目、中央 banner 与商品楼层、右侧用户信息与快捷区。
现在它仍然直接拿商品列表拼装，能跑，但不适合继续扩展。

建议补：

### 2.1 首页装修数据

- `GET /home/config`

建议返回：

```json
{
  "banners": [
    {
      "id": 1,
      "title": "热销Top榜",
      "subtitle": "火热抢购中",
      "image": "https://...",
      "targetType": "product",
      "targetId": 1001
    }
  ],
  "categories": [
    {
      "id": 1,
      "name": "每日必抢",
      "icon": "https://..."
    }
  ],
  "floors": [
    {
      "id": 1,
      "title": "今日热卖",
      "type": "product_list",
      "productIds": [1, 2, 3, 4]
    }
  ]
}
```

作用：

- 首页 banner 不再写死
- 分类宫格不再写死
- 商品楼层可后台配置
- 可支持网页版首页的楼层化运营

### 2.1.1 首页频道导航

- `GET /home/channels`

建议返回：

```json
[
  {
    "id": 1,
    "name": "热销榜",
    "path": "/app/home"
  },
  {
    "id": 2,
    "name": "闪购",
    "path": "/app/flash"
  }
]
```

作用：

- 顶部频道栏不再写死
- 可以像淘宝网页版一样动态调整导航栏目

### 2.2 商品分类

- `GET /product/categories`
- `GET /product/products?categoryId=...`
  - 说明：该接口保留给后续普通商品列表，当前秒杀商品已独立到 `/seckill-product/products`

作用：

- 首页分类区可点
- 首页与搜索页可按分类筛选

### 2.2.1 热门搜索词 / 搜索联想

- `GET /search/hot-keywords`
- `GET /search/suggest?keyword=...`

作用：

- 顶部搜索区不再写死热门词
- 输入商品关键词时可以做联想提示

### 2.3 商品详情增强

当前 `GET /seckill-product/products/{id}` 字段太少。

建议补充字段：

- `mainImage`
- `images`
- `description`
- `sales`
- `tags`
- `categoryId`
- `categoryName`

否则详情页只能展示价格和时间，信息不够。

## 3. 闪购页还建议补的接口

当前秒杀功能主流程已经有了，但还缺“运营化”和“可视化”数据。

### 3.1 秒杀活动列表

- `GET /seckill/activities`

建议返回：

```json
[
  {
    "id": 1,
    "title": "午间场",
    "startTime": "2026-03-17T12:00:00",
    "endTime": "2026-03-17T14:00:00",
    "status": "RUNNING"
  }
]
```

作用：

- 支持“上午场 / 下午场 / 晚间场”
- 秒杀页顶部可以按场次切换

### 3.2 某场次商品

- `GET /seckill/activities/{activityId}/products`

作用：

- 闪购页不必再和首页共用一个商品列表
- 便于后续支持多场活动

### 3.3 秒杀结果详情

当前 `GET /seckill/result/{productId}` 只有 `status/message/orderId`。

建议补充：

- `queuePosition`
- `estimatedWaitMs`
- `productId`
- `finalStatus`

作用：

- 前端可以更像真实秒杀产品
- 排队状态更清晰

## 4. 购物车页必须补的接口

当前购物车是前端本地草案，不是真实购物车。

最少要补：

### 4.1 查询购物车

- `GET /cart/items`

### 4.2 加入购物车

- `POST /cart/items`

请求体示例：

```json
{
  "productId": 1,
  "quantity": 2
}
```

### 4.3 修改数量

- `PUT /cart/items/{itemId}`

### 4.4 删除购物车项

- `DELETE /cart/items/{itemId}`

### 4.5 清空购物车

- `DELETE /cart/items`

建议购物车返回字段包含：

- `itemId`
- `productId`
- `productName`
- `mainImage`
- `price`
- `seckillPrice`
- `quantity`
- `stock`
- `selected`

如果你后面还想做真实下单，还需要：

- `POST /order/checkout`

## 5. “我的”页面必须补的接口

当前“我的”页只能展示：

- 登录返回的 `userId`
- 登录返回的 `username`
- 我的订单

这离完整个人中心还差很多。

### 5.1 当前用户信息

- `GET /auth/me`

建议返回：

```json
{
  "userId": 1,
  "username": "neo",
  "nickname": "Neo",
  "avatar": "https://...",
  "mobile": "13800000000",
  "email": "neo@example.com"
}
```

### 5.2 修改个人资料

- `PUT /auth/me`

### 5.3 地址列表

- `GET /user/addresses`
- `POST /user/addresses`
- `PUT /user/addresses/{id}`
- `DELETE /user/addresses/{id}`

### 5.4 订单分页和状态筛选

当前订单接口没有分页。

建议改造或新增：

- `GET /order/orders?pageNum=1&pageSize=10&status=0`

作用：

- “我的订单”可以做分页
- 可以做“待支付 / 已支付 / 已取消”筛选

### 5.5 订单详情增强

当前订单详情字段也比较少。

建议补：

- `productName`
- `productImage`
- `quantity`
- `totalAmount`
- `payAmount`
- `payStatus`
- `addressInfo`

## 7. 推荐优先级

如果你想按投入产出比推进，建议顺序如下：

1. `GET /auth/me`
2. 购物车整套接口 `GET/POST/PUT/DELETE /cart/items`
3. 订单分页与状态筛选
4. 首页装修接口 `GET /home/config`
5. 秒杀活动场次接口
6. 地址、收藏、推荐等增强能力

---

## 8. 前端当前实现和后端缺口的对应关系

- 首页目前使用商品列表直接拼装：缺首页装修接口
- 顶部频道栏和热门词目前写死：缺频道导航和搜索推荐接口
- 闪购页目前直接使用商品列表筛选：缺秒杀活动维度接口
- 购物车目前是前端本地数据：缺购物车接口
- 我的页面目前个人信息来自登录态：缺 `/auth/me`
- 订单页目前无分页：缺分页查询接口

如果你后续开始补接口，前端这版结构可以继续接，不需要推翻重做。
