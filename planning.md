# 当前阶段规划与后续业务路线

> 本文档基于当前仓库真实实现整理，目标是说明“现在已经做到哪里”和“后面最值得继续做什么”。

## 1. 当前项目阶段

项目已经从早期的“秒杀原型”演进成“可联调的轻量商城系统”。

当前有两条核心业务线：
- 普通商品链路
- 秒杀商品链路

两条链路都已经具备基础闭环，但在商城化、运营化和订单生命周期上仍有继续完善空间。

## 2. 当前已经完成的能力

### 2.1 认证与网关
- 网关统一路由
- JWT 鉴权
- 方法级白名单放行
- Bearer Token 统一透传
- `X-User-Id` 下游身份透传
- 登录
- 注册
- 获取当前用户
- 修改密码

### 2.2 普通商品业务
- 普通商品列表
- 普通商品详情
- 购物车增删改查
- 商品卡片“立即购买”
- 独立结算页
- 基于已选购物车商品创建普通订单
- 普通订单列表
- 普通订单详情
- 普通订单模拟支付
- 普通订单支付状态查询

### 2.3 秒杀业务
- 秒杀商品列表
- 秒杀商品详情
- 发起秒杀
- 秒杀结果轮询
- Redis 秒杀结果缓存
- RabbitMQ 异步创建秒杀订单
- 秒杀订单列表
- 秒杀订单详情
- 秒杀订单模拟支付
- 秒杀订单支付状态查询

### 2.4 前端页面
- 登录 / 注册页
- 商城首页
- 秒杀会场页
- 购物车页
- 独立结算页
- 个人中心页
- 商品详情抽屉
- 订单详情弹窗

## 3. 当前已经具备的业务闭环

### 3.1 普通商品闭环
1. 登录
2. 浏览普通商品
3. 查看商品详情
4. 加入购物车或立即购买
5. 进入独立结算页
6. 创建普通订单
7. 模拟支付
8. 查看订单支付状态

### 3.2 秒杀商品闭环
1. 登录
2. 浏览秒杀商品
3. 查看商品详情
4. 发起秒杀
5. 轮询秒杀结果
6. 查看秒杀订单详情
7. 模拟支付
8. 查看支付状态

## 4. 当前仍未完善的业务能力

下面这些不是“代码优化项”，而是下一阶段最值得继续做的真实业务功能。

### 4.1 地址管理

当前问题：
- 结算页仍然通过手填 `addressSnapshot` 传递收货信息
- 能跑通，但不够像完整商城

建议补齐：
- 地址列表
- 新增地址
- 编辑地址
- 删除地址
- 默认地址
- 结算页地址选择

### 4.2 购物车结算预览

当前问题：
- 结算金额和商品可支付状态还缺少一个统一的后端预览接口

建议补齐：
- 结算预览接口
- 已选商品汇总
- 库存不足与失效商品提示
- 后端统一金额校准

### 4.3 批量全选 / 取消全选

当前问题：
- 单项更新已具备
- 但购物车底部的全选更适合有批量接口支撑

建议补齐：
- 批量选中
- 批量取消选中

### 4.4 订单取消 / 超时关闭

当前问题：
- 普通订单支持创建和模拟支付
- 但缺少“未支付取消”和生命周期继续流转

建议补齐：
- 取消订单
- 超时自动关闭
- 状态流转完善

### 4.5 统一订单中心

当前问题：
- 个人中心仍偏前端聚合普通订单和秒杀订单

建议补齐：
- 统一订单聚合接口
- 订单分页
- 按状态筛选
- 按订单类型筛选

### 4.6 首页运营配置

当前问题：
- 首页轮播、热词、频道更偏前端静态配置

建议补齐：
- 首页配置接口
- 热门搜索接口
- 频道配置接口
- 推荐商品接口

### 4.7 秒杀运营增强

当前问题：
- 秒杀链路能跑，但运营感还不够强

建议补齐：
- 秒杀场次
- 场次商品列表
- 排队位置
- 预计等待时间

## 5. 推荐优先级

### P0：优先补成更完整的商城
- 地址管理
- 购物车结算预览
- 批量全选 / 取消全选
- 订单取消 / 超时关闭

### P1：提升用户体验与页面完整度

- 首页运营配置

### P2：继续增强秒杀与运营能力

- 排队信息增强
- 更完整的活动展示

## 6. 推荐开发顺序

### 第一阶段：把普通商城链路做实
1. 地址管理
2. 购物车结算预览
3. 批量全选 / 取消全选
4. 普通订单取消 / 超时关闭

### 第二阶段：把个人中心与首页做完整
1. 统一订单中心
2. 首页运营接口
3. 热门搜索与搜索建议

### 第三阶段：把秒杀运营感做强
1. 秒杀场次
2. 场次维度商品列表
3. 排队位置与等待时间

## 8. 当前需要长期记住的约束

- 所有前端请求统一走 Gateway
- 首页与秒杀页可匿名访问，但购物车、个人中心、结算页必须登录
- 商品详情点击前已有前端登录拦截
- 普通订单创建不再由前端直接传 `items`
- 普通商品链路是 `product-service -> OpenFeign -> order-service`
- 秒杀链路仍是 `Redis/Lua + RabbitMQ + order-service`

## 9. 下一步最推荐做什么

如果只选一个方向继续推进，我最推荐：

1. 地址管理
2. 购物车结算预览

这两块补完后，普通商品这条链路会从“能跑通”升级成“更像真正商城”。
